## 2. 検出された問題点

### 2.1 バグ・ロジックエラー

#### ✅ ~~🔴 BUG-001: 再帰の深さ制限がない~~ **→ 修正済**
- **ファイル**: `src/interpreter/evaluator.ts` — `evaluate()` 関数
- **重要度**: Critical
- **対応**: MAX_DEPTH=1000 の再帰深度制限を evaluate/evaluateInner パターンで実装
- **内容**: `(defun f (x) (f x)) (f 0)` のような無限再帰でブラウザのスタックオーバーフローが発生する。`LOOP` には `MAX_ITERATIONS` の制限があるが、再帰呼び出しには制限がない。
- **影響**: ブラウザタブがクラッシュする
- **修正案**: グローバルな再帰深度カウンタを追加し、1000レベルを超えたらエラーをスロー

#### 🟠 BUG-002: MOD のゼロ除算チェックがない
- **ファイル**: `src/interpreter/evaluator.ts` — `MOD` ビルトイン
- **重要度**: Major
- **内容**: `(mod 10 0)` を実行すると `NaN` が返り、エラーにならない
- **修正案**: `if (args[1].value === 0) throw new Error('mod: ゼロで割ることはできません');` を追加

#### 🟠 BUG-003: NTH の負のインデックスが未検証
- **ファイル**: `src/interpreter/evaluator.ts` — `NTH` ビルトイン
- **重要度**: Major
- **内容**: `(nth -1 '(1 2 3))` で `undefined` が返りエラーにならない。Common Lisp では負インデックスはエラー
- **修正案**: インデックスの範囲チェックを追加

#### 🟠 BUG-004: SUBSEQ がリストに未対応
- **ファイル**: `src/interpreter/evaluator.ts` — `SUBSEQ` ビルトイン
- **重要度**: Major
- **内容**: 文字列のみ対応。Common Lisp ではリストにも使用可能
- **修正案**: `list` 型の分岐を追加し `elements.slice()` で対応

#### 🟠 BUG-005: MAPCAR が複数リスト引数に未対応
- **ファイル**: `src/interpreter/evaluator.ts` — `MAPCAR` ビルトイン
- **重要度**: Major
- **内容**: `(mapcar #'+ '(1 2) '(3 4))` が動作しない。引数を1リストのみ受け付ける
- **修正案**: `args.slice(1)` を複数リストとして zip 処理

#### 🟡 BUG-006: FORMAT の対応ディレクティブが限定的
- **ファイル**: `src/interpreter/evaluator.ts` — `FORMAT` ビルトイン
- **重要度**: Minor
- **内容**: `~A`, `~S`, `~D`, `~%`, `~~` のみ対応。`~R`, `~B`, `~X` 等は未実装
- **影響**: 学習用途では十分だが、上級者向け問題で制限になる可能性

### 2.2 セキュリティ

#### ✅ ~~🔴 SEC-001: スタックオーバーフロー保護なし~~ **→ 修正済**
- **ファイル**: `src/interpreter/evaluator.ts`
- **重要度**: Critical
- **内容**: BUG-001 と同根。悪意あるコード or 学習者の誤りでブラウザクラッシュ
- **対応**: BUG-001 と同時に修正済
- **修正案**: 
  ```typescript
  let evaluationDepth = 0;
  const MAX_DEPTH = 1000;
  function evaluate(expr, env, output) {
    if (++evaluationDepth > MAX_DEPTH)
      throw new Error('再帰が深すぎます（最大深度を超えました）');
    try { /* 既存処理 */ } 
    finally { evaluationDepth--; }
  }
  ```

#### ✅ ~~🔴 SEC-002: 出力バッファのサイズ制限なし~~ **→ 修正済**
- **ファイル**: `src/interpreter/index.ts` — `executeLisp`
- **重要度**: Critical
- **対応**: MAX_OUTPUT_SIZE=1MB の出力バッファ制限を実装
- **内容**: `(dotimes (i 100000) (print i))` で数MB の文字列が生成されメモリ圧迫
- **修正案**: 出力バッファに最大サイズ（例: 1MB）を設定し超過時にエラー

#### ✅ ~~🟠 SEC-003: 同期実行による UI ブロック~~ **→ 修正済**
- **ファイル**: `src/pages/EditorPage.tsx`, `src/worker/`
- **重要度**: Major
- **対応**: Web Worker によるバックグラウンド実行 + 10秒タイムアウトを実装
- **内容**: `executeLisp()` が同期関数のため、重い処理中は UI が完全にフリーズ
- **修正案**: Web Worker でインタプリタを実行、タイムアウト付き

#### 🟠 SEC-004: Markdown レンダリングの XSS リスク（潜在的）
- **ファイル**: `src/components/ProblemView.tsx` — `SimpleMarkdown`
- **重要度**: Major（将来的にリスク）
- **内容**: 現在問題データはハードコードのため安全。将来 API 経由で問題データを取得する場合、`dangerouslySetInnerHTML` は使っていないが、JSX 内での文字列直接展開に注意が必要
- **修正案**: 問題データの外部化時に DOMPurify 等のサニタイズライブラリを導入

### 2.3 TypeScript 型安全性

#### 🟠 TS-001: ビルトイン関数の型定義不整合
- **ファイル**: `src/interpreter/evaluator.ts`
- **重要度**: Major
- **内容**: `defBuiltin` のコールバックで `callEnv` パラメータを追加受け取りしている高階関数（`MAPCAR` 等）があるが、`LispFunction` の `fn` 型定義では第2引数が `Environment` のため暗黙的に互換性があるものの、意図が不明確
- **修正案**: 高階関数用の型を分離するか、明示的にオーバーロード

#### 🟡 TS-002: ループ内の配列インデックスアクセスの型安全性
- **ファイル**: `src/interpreter/evaluator.ts`
- **重要度**: Minor
- **内容**: `args[i]` のループ内アクセスで TypeScript のナローイングが効かないパターンがある（既に修正済みの `-` `/` を除く）
- **修正案**: ループ変数を一旦ローカル変数に代入するパターンを統一

#### 🟠 ACC-003: カラーコントラスト不足の可能性

## 3. アーキテクチャ評価

### 現在のアーキテクチャ

```
[ユーザーアクセス]
     ↓
[HashRouter (react-router-dom)]
     ├─ / → [LearnPage] → [LispGuide] / [ProblemView]
     └─ /editor → [EditorPage]
                       ↓
              [CodeMirror エディタ]
                       ↓
              [Parser (tokenize → parse)]
                       ↓
              [Evaluator (evaluate + builtins)]
                       ↓
              [OutputPanel (結果表示)]
```

### 良い点 ✅

1. **完全クライアントサイド** — サーバー依存なし、デプロイが容易
2. **2ページ構成** — 学習ページとエディタページの分離で関心の分離を実現
3. **コンポーネント分離** — Editor / Output / Problem / Guide が明確に分離
4. **インタプリタの独立性** — React に依存せず、純粋な TypeScript
5. **問題データの宣言的定義** — TypeScript の型安全性を活用
6. **包括的テスト** — インタプリタ単体 + UIコンポーネントテスト 278件

## 6. デプロイ・運用上の注意

### バンドルサイズ

| ファイル | サイズ | Gzip |
|---------|--------|------|
| index.html | 0.61 KB | 0.43 KB |
| index.css | 6.57 KB | 1.76 KB |
| index.js | 567.30 KB | 183.75 KB |

⚠️ JS バンドルが 500KB を超えています。主因は CodeMirror です。

**改善案**:
- CodeMirror の動的インポート (`React.lazy`)
- 不要な CodeMirror 拡張の除去
- `build.rollupOptions.output.manualChunks` でコード分割

### ブラウザ互換性

- ES2020 ターゲットのため、IE11 非対応
- Chrome 80+, Firefox 80+, Safari 14+, Edge 80+ で動作

### 注意事項

1. `React.StrictMode` が有効（開発時の二重レンダリングに注意）
2. `.gitignore` が未作成 → `node_modules/`, `dist/` を除外すること
3. `package-lock.json` をバージョン管理に含めること
