## 3. 実施方法

### 3.1 デプロイ確認

- GitHub CLI で `Build & Deploy` workflow の success を確認
- GitHub Pages API で公開 URL と公開方式が workflow ベースであることを確認

### 3.2 ルート到達性確認

- 公開 URL のトップ画面を取得し、home 画面の主要文言を確認
- `/#/problems`, `/#/editor`, `/#/repl` を直接取得し、各ページ固有の文言が表示されることを確認

### 3.3 公開後スモークテスト

Playwright + Chromium の headless 実行で、次の操作を順に確認した。

1. トップページを開く
2. 問題一覧の最初の問題を選択する
3. 解答を表示する
4. 「エディタで解く」でエディタページへ遷移する
5. 解答コードを実行し、正解バッジと実行結果を確認する
6. REPL ページへ移動する
7. `(+ 1 2 3)` を評価して `6` を確認する
8. クリア操作後に初期表示へ戻ることを確認する

### 3.4 localStorage 保存・復元確認

同じく公開 URL 上で、次の保存と復元を確認した。

1. `localStorage.clear()` 後に最初の問題 `basic-01` を選択する
2. 解答を表示し、エディタで正答コードを実行する
3. `lisp-playground-problem-id` が `basic-01` として保存されることを確認する
4. `lisp-playground-solved-problem-ids` に `basic-01` が保存されることを確認する
5. `lisp-playground-code` に解答コード `(print (* 5 6))` を含むことを確認する
6. `/#/editor` を再読込し、現在の問題ラベルとエディタ内容が復元されることを確認する
7. `/` を開き直し、学習ページでも選択中問題が復元されることを確認する

### 3.5 問題データ総当たり確認

同じく公開 URL 上で、問題一覧に表示される全 38 問について次を順に確認した。

1. 問題を選択する
2. 解答を表示する
3. エディタページへ遷移する
4. 解答コードを実行する
5. `✓ 正解！` バッジが表示され、`✗ 不正解` や実行エラーが出ないことを確認する

結果は 38/38 問が通過し、fail=0 だった。

### 3.6 モバイル表示確認

公開 URL 上で Playwright + Chromium を使い、390px 幅と 320px 幅の 2 パターンを確認した。

1. `/` を開き、学習ページのヘッダー、問題導線、フリーモード導線が viewport 内に収まることを確認する
2. `/#/editor` を開き、ヘッダー、実行ボタン、出力領域が横スクロールなしで表示されることを確認する
3. `/#/repl` を開き、ナビゲーション、入力領域、実行結果表示が viewport 内に収まることを確認する

結果は 390px / 320px の両方で学習ページ、エディタ、REPL が通過した。

### 3.7 home 画面と問題一覧ページ導線確認

`29c898a` の再デプロイ後、公開 URL 上で Playwright + Chromium を使い、home 画面と問題一覧ページの新しい導線を確認した。

1. `/#/problems` を開き、ヘッダーロゴから `#/` の home 画面へ戻れることを確認する
2. home 画面を最下部までスクロールし、下部 CTA が 390px / 320px 幅でも見切れず到達できることを確認する
3. `/#/problems` から最初の問題詳細へ進み、エディタへ移動したあと `← 問題一覧に戻る` で `/#/problems` に戻れることを確認する
4. 390px / 320px 幅の両方で横方向オーバーフローが発生しないことを確認する

結果は 390px / 320px の両方で通過した。
